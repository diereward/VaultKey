# VaultKey

VaultKey — локальный менеджер паролей для Android. Приложение хранит данные на устройстве, шифрует пароли и поддерживает вход по мастер-паролю или биометрии.

## Возможности

- Хранение записей с названием, логином, паролем, URL, заметками, категорией и аватаром.
- Генератор паролей с настройкой длины и наборов символов.
- Оценка надежности пароля через zxcvbn.
- Поиск записей по названию, логину и URL.
- Темная, светлая и системная тема.
- Автоблокировка при бездействии.
- Очистка буфера обмена через 30 секунд после копирования.
- Загрузка favicon по URL напрямую с сайта через `https://domain/favicon.ico`.
- Ручной выбор аватара из документов Android с постоянным разрешением на чтение.

## Безопасность

- Пароли записей шифруются AES-GCM.
- Новые записи используют формат `v4` с ключом, полученным через PBKDF2-HMAC-SHA256 с 120 000 итераций.
- Мастер-пароль проверяется через PBKDF2-verifier и хранится через Android Keystore.
- Старые legacy-хранилища поддерживаются для миграции.
- Старые записи формата `v3` при успешном открытии автоматически перешифровываются в `v4`.
- Ошибки расшифровки не записываются в поля пароля и показываются пользователю как ошибка.
- Облачный backup и перенос данных отключены через `backup_rules.xml` и `data_extraction_rules.xml`.

## Сеть

Приложению нужен доступ в интернет только для автоматической загрузки favicon по URL. Запрос идет напрямую к домену записи, без сторонних favicon API. Если favicon не найден или сеть недоступна, показывается первая буква названия.

## Сборка

```powershell
$env:JAVA_HOME='C:\Program Files\Android\Android Studio\jbr'
$env:Path="$env:JAVA_HOME\bin;$env:Path"
./gradlew.bat assembleDebug
```

Готовый debug APK создается в `app/build/outputs/apk/debug/app-debug.apk`.
